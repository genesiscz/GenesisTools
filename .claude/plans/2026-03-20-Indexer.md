# General-Purpose Semantic Indexer — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build `tools indexer` — a Cursor-inspired semantic code search engine with AST-aware chunking, multi-provider embeddings, hybrid BM25+vector+RRF search, and an event-driven architecture for CLI/daemon/programmatic use.

**Architecture:** tree-sitter AST chunking (@ast-grep/napi) → Embedder (darwinkit/HF/cloud) → FTS5+vector storage (SQLite primary, optional Orama hot cache, optional Turbopuffer cloud) → hybrid BM25+cosine+RRF retrieval. Change detection uses git-first + chunk-level Merkle tree. Fully typed event system for progress tracking.

**Tech Stack:** Bun, @ast-grep/napi (Rust N-API tree-sitter), SQLite FTS5 (bun:sqlite), @orama/orama, @turbopuffer/turbopuffer, chokidar, commander, @clack/prompts

The typed event system is the foundation — every other component emits events through it.

```typescript
// ─── Base timestamp (epoch ms, injected by emit) ────────────────
type Ts = { ts: number };

// ─── Event map: single source of truth ──────────────────────────
export interface IndexerEventMap {
    "scan:start":      Ts & { indexName: string; strategy: string };
    "scan:complete":   Ts & { indexName: string; added: number; modified: number; deleted: number; unchanged: number };

    "chunk:file":      Ts & { indexName: string; filePath: string; chunks: number; parser: "ast" | "line" };
    "chunk:skip":      Ts & { indexName: string; filePath: string; reason: "unchanged" | "ignored" | "binary" };

    "embed:start":     Ts & { indexName: string; totalChunks: number; provider: string; dimensions: number };
    "embed:progress":  Ts & { indexName: string; completed: number; total: number; currentFile: string };
    "embed:complete":  Ts & { indexName: string; embedded: number; skipped: number; durationMs: number };

    "sync:start":      Ts & { indexName: string; mode: "incremental" | "full" };
    "sync:complete":   Ts & { indexName: string; durationMs: number; stats: SyncStats };
    "sync:error":      Ts & { indexName: string; error: string; filePath?: string };

    "watch:start":     Ts & { indexName: string; strategy: string };
    "watch:change":    Ts & { indexName: string; filePath: string; event: "add" | "modify" | "delete" };
    "watch:stop":      Ts & { indexName: string };

    "search:query":    Ts & { indexName: string; query: string; mode: string; results: SearchResult<Record<string, unknown>>[]; resultsCount: number; durationMs: number; cached: boolean };
}

// ─── Derived types ──────────────────────────────────────────────
export type EventName = keyof IndexerEventMap;
export type Namespace = EventName extends `${infer NS}:${string}` ? NS : never;
export type EventsIn<NS extends Namespace> = Extract<EventName, `${NS}:${string}`>;

export type WildcardPayload<NS extends Namespace> = {
    [K in EventsIn<NS>]: IndexerEventMap[K] & { event: K };
}[EventsIn<NS>];

// ─── Inline callbacks derived from event map ────────────────────
type CamelEvent<S extends string> =
    S extends `${infer NS}:${infer A}` ? `on${Capitalize<NS>}${Capitalize<A>}` : never;

export type IndexerCallbacks = {
    [K in EventName as CamelEvent<K>]?: (payload: IndexerEventMap[K]) => void;
};

// ─── Emitter interface ──────────────────────────────────────────
export interface IndexerEmitter {
    on<K extends EventName>(event: K, handler: (payload: IndexerEventMap[K]) => void): this;
    on<NS extends Namespace>(event: `${NS}:*`, handler: (payload: WildcardPayload<NS>) => void): this;
    on(event: "*", handler: (payload: WildcardPayload<Namespace>) => void): this;
    off<K extends EventName>(event: K, handler: (payload: IndexerEventMap[K]) => void): this;
    emit<K extends EventName>(event: K, payload: Omit<IndexerEventMap[K], "ts">): void;
}

export interface SyncStats {
    filesScanned: number;
    chunksAdded: number;
    chunksUpdated: number;
    chunksRemoved: number;
    chunksUnchanged: number;
    embeddingsGenerated: number;
    durationMs: number;
}
```

**Implementation:** `IndexerEventEmitter` class implementing `IndexerEmitter`:
- Store handlers in `Map<string, Set<Function>>`
- `emit()` auto-injects `ts: Date.now()`
- Wildcard matching: `"embed:*"` matches any key starting with `"embed:"`
- Also dispatch to any inline `IndexerCallbacks` registered for the operation
- Fire `"*"` handlers for every event

**Tests:**
- `.on("scan:start", handler)` fires handler with correct payload
- `.on("embed:*", handler)` fires for `embed:start`, `embed:progress`, `embed:complete`
- `.on("*", handler)` fires for everything
- `.off()` removes handler
- `emit()` auto-injects `ts` as a number
- Wildcard payload includes `event` discriminant field
- Type-level test: verify `IndexerCallbacks` has the expected keys

**Commit:** `feat(indexer): add typed event system with wildcard support`

## Task 3: Chunker — AST-aware code splitting (`src/indexer/lib/chunker.ts`)

**Files:**
- Create: `src/indexer/lib/chunker.ts`
- Create: `src/indexer/lib/chunker.test.ts`

**Dependencies:** `bun add @ast-grep/napi`

**Implementation:**

```typescript
export interface ChunkResult {
    chunks: ChunkRecord[];
    language: string | null;
    parser: "ast" | "line";
}

export function chunkFile(opts: {
    filePath: string;
    content: string;
    strategy: "ast" | "line" | "auto";
    maxTokens?: number;
}): ChunkResult;
```

**AST strategy (`@ast-grep/napi`):**
- Map file extension → `Lang` enum (`.ts` → `Lang.TypeScript`, `.tsx` → `Lang.Tsx`, etc.)
- Parse with `parse(lang, content)`
- Extract semantic blocks via `root.findAll({ rule: { kind: "function_declaration" } })` etc.
- Language-specific node kinds:
  - **TS/JS:** `function_declaration`, `arrow_function`, `class_declaration`, `method_definition`, `interface_declaration`, `type_alias_declaration`, `export_statement`
  - **Python:** `function_definition`, `class_definition`, `decorated_definition`
  - **Go:** `function_declaration`, `method_declaration`, `type_declaration`
  - **Rust:** `function_item`, `impl_item`, `struct_item`, `enum_item`
- For each node: extract `{ content: node.text(), startLine: node.range().start.line, endLine: node.range().end.line, kind, name }`
- If a chunk exceeds `maxTokens`, split into sub-chunks at line boundaries
- Generate `id` as `SHA-256(content)` — this IS the Merkle leaf hash

**Line strategy (fallback):**
- Split at double newlines (paragraph boundaries)
- Merge small chunks up to `maxTokens`
- Used for markdown, plain text, unknown file types

**Heading strategy (markdown):**
- Split at `##` / `###` headings — each section = one chunk
- Include the heading text as the chunk `name`
- Merge consecutive paragraphs under the same heading
- If a section exceeds `maxTokens`, split at paragraph boundaries

**Message strategy (email, chat):**
- Each message/email = one chunk
- Content: `"Subject: {subject}\nFrom: {sender}\nTo: {recipients}\n\n{body}"`
- Metadata fields (date, sender, recipients) stored separately for filtering
- `kind: "message"`, `name: subject` (or first line of chat message)

**JSON strategy:**
- Top-level array → each element = one chunk, content = serialized element
- Top-level object → each key with value = one chunk, `name: keyPath`
- Nested objects: flatten to dot-notation paths (e.g. `"users[0].profile"`)
- Uses `SafeJSON.stringify()` for content, `SHA-256(content)` for chunk ID
- Unchanged elements produce identical hashes → no re-embedding

**Auto strategy:**
- Code index: `ast` for known languages, `heading` for `.md`, `json` for `.json`, `line` for rest
- Mail/chat index: `message`
- Files index: `heading` for `.md`, `json` for `.json`, `line` for rest

**Tests:**
- TypeScript function extraction produces correct chunks with names
- Class with methods produces parent + child chunks
- Markdown falls back to line chunking
- Chunks respect maxTokens limit
- SHA-256 hash is deterministic (same content = same id)
- Unknown extensions use line strategy

**Commit:** `feat(indexer): add AST-aware chunker with @ast-grep/napi`

```typescript
export function buildMerkleTree(opts: {
    baseDir: string;
    files: Array<{ path: string; chunkHashes: string[] }>;
}): MerkleNode;

export function diffMerkleTrees(opts: {
    previous: MerkleNode | null;
    current: MerkleNode;
}): {
    added: string[];      // file paths
    modified: string[];   // file paths (any chunk hash changed)
    deleted: string[];    // file paths
    unchanged: string[];  // file paths
};
```

- Directory hash = SHA-256 of sorted children hashes
- File hash = SHA-256 of sorted chunk hashes
- `diffMerkleTrees` walks both trees, comparing hashes top-down
- When a directory hash matches → skip entire subtree (this is the O(log n) win)
- When a directory hash differs → recurse into children
- Store/load Merkle tree as JSON in `~/.genesis-tools/indexer/<name>/merkle.json`

**Tests:**
- Identical trees produce no diff
- Added file detected
- Modified file detected (changed chunk hash)
- Deleted file detected
- Moved file (same content, new path): shows as delete + add, but chunk hashes match so no re-embedding needed
- Directory-level short-circuit: unchanged subtree skips all children

```typescript
    config: IndexConfig;
    previousMerkle: MerkleNode | null;
    currentChunks: Map<string, string[]>;  // filePath → chunkHashes
}): Promise<DetectedChanges>;
```

**Strategies:**
- **`git`:** Use `createGit()` from `src/utils/git/core.ts`. Run `git diff --name-status HEAD~1` (or since last indexed commit SHA). Respects `.gitignore` automatically. Fast for repos with few changes.
- **`merkle`:** Compare Merkle trees. Content-addressed — immune to moves/renames. Works on non-git dirs.
- **`git+merkle`:** Git diff tells us which files changed (fast). For those files, Merkle tree at chunk level determines which chunks actually changed (avoids re-embedding unchanged chunks in modified files). Best of both worlds.
- **`chokidar`:** Real-time filesystem events (for watch mode only, not initial scan).

**Tests:**
- Git strategy detects added/modified/deleted files
- Merkle strategy detects content changes regardless of path
- git+merkle: git finds modified files, merkle identifies unchanged chunks within them
- Non-git directory falls back gracefully

## Task 7: Indexer Core — the main pipeline (`src/indexer/lib/indexer.ts`)

**Files:**
- Create: `src/indexer/lib/indexer.ts`
- Create: `src/indexer/lib/indexer.test.ts`

This is the heart — orchestrates scan → chunk → hash → embed → store.

```typescript
export class Indexer extends IndexerEventEmitter {
    static async create(config: IndexConfig): Promise<Indexer>;

    get name(): string;
    get config(): IndexConfig;
    get stats(): IndexStats;

    async reindex(callbacks?: IndexerCallbacks): Promise<SyncStats>;
    async sync(callbacks?: IndexerCallbacks): Promise<SyncStats>;
    async search(query: string, opts?: SearchOptions & IndexerCallbacks): Promise<SearchResult<ChunkRecord>[]>;

    startWatch(callbacks?: IndexerCallbacks): void;
    stopWatch(): void;

    async close(): Promise<void>;
}
```

**`sync()` pipeline:**
1. Emit `scan:start` → run `detectChanges()` → emit `scan:complete`
2. For added/modified files: emit `chunk:file` → run `chunkFile()` per file
3. For unchanged files: emit `chunk:skip`
4. Hash chunks → diff against stored Merkle → identify chunks needing embedding
5. Emit `embed:start` → batch `embedder.embedMany()` with progress → emit `embed:progress` → emit `embed:complete`
6. Store chunks + embeddings via `IndexStore` → emit `sync:complete`
7. Inline callbacks fire alongside event emitter events

**`reindex()`:** Same as `sync()` but ignores Merkle / git diff — processes everything.

**`search()`:** Delegates to `IndexStore.search()`, emits `search:query` with full results, logs to `search_log`.

**`startWatch()`:** Uses chokidar or polling (based on config). On changes, calls `sync()` automatically.

**Tests (with temp directory + fixture files):**
- `sync()` indexes new files and produces correct stats
- `sync()` skips unchanged files on second run
- `sync()` detects modified files and re-indexes only changed chunks
- `reindex()` processes everything regardless
- `search()` returns ranked results with chunk metadata
- Events fire in correct order during sync
- Inline callbacks receive correct payloads

**Commit:** `feat(indexer): add core Indexer pipeline with sync/reindex/search`

```
  --name <name>        Index name (default: directory basename)
  --type <type>        Index type: code, files, mail, chat (default: auto-detect)
```

```
Options:
```
